## 1. 系統目標

提供單一使用者一套「AI 輔助的股票分析與模擬交易」工具：
1. 呈現台股/美股個股走勢與技術指標
2. 由免費 AI（Gemini 系列）產生分析報告、買賣建議與情境推演
3. 比較多檔股票的報酬率與 ETF 折溢價
4. AI 依建議在虛擬帳戶自動模擬買賣，追蹤績效以驗證 AI 建議品質

**成本約束（硬性）**：全系統營運成本 = 0（免費資料源、免費 AI 額度、免費或本機部署）。

## 2. 使用者與角色

| 角色 | 說明 |
|------|------|
| 投資人（唯一使用者） | 系統擁有者本人，具基本投資知識；無多使用者、無權限分級 |
| 系統排程（背景角色) | 每日自動執行資料更新、AI 批次分析、模擬下單 |

## 3. 使用案例（Use Cases）

```
UC-01 切換市場        使用者以 Radio Button 切換台股/美股，全站資料池隨之切換
UC-02 查看個股走勢    搜尋/點選個股 → K 線圖＋技術指標＋成交量
UC-03 查看 AI 分析    個股頁查看當日 AI 報告（動作/信心/目標價/三情境/風險）
UC-04 觸發深度分析    使用者手動要求重新深度分析（消耗 3.5 Flash 額度，20 RPD 上限）
UC-05 比較報酬率      勾選多檔 → 報酬率表格（日/週/月/YTD/年化）＋正規化折線圖
UC-06 查看折溢價      ETF 折溢價列表排序＋單檔歷史折溢價走勢
UC-07 管理自選股      新增/移除自選（依市場隔離）
UC-08 設定 AI 託管    指定哪些股票交給 AI 模擬操作、每檔部位上限
UC-09 查看模擬績效    權益曲線、持倉、交易日誌（每筆可展開 AI 決策理由）
UC-10 查看預測走勢    個股圖上疊加量化預測區間帶＋AI 三情境目標價
UC-B1 (排程)每日資料更新   收盤後抓日線/法人/NAV → 算指標 → 算折溢價
UC-B2 (排程)每日 AI 批次   flash-lite 批次分析託管清單 → 產生報告
UC-B3 (排程)AI 模擬下單    3.5 優先產生交易報告（可降級）＋風控規則產生模擬委託 → 隔日開盤價成交
UC-B4 (排程)新聞研究       Antigravity 對自選股搜尋近期新聞產出摘要
```

## 4. 功能需求（FR）

| 編號 | 需求 | 優先級 |
|------|------|--------|
| FR-01 | 台股/美股雙市場，所有功能依市場隔離（股票池、自選、模擬帳戶、幣別） | P0 |
| FR-02 | 日線 OHLCV 走勢圖（K 線/折線切換）、MA/KD/MACD/RSI/布林指標疊圖 | P0 |
| FR-03 | AI 結構化分析報告：action、confidence、目標價區間、停損、理由、三情境、風險 | P0 |
| FR-04 | 多股報酬率比較表（可排序）＋基準日=100 的正規化折線圖 | P0 |
| FR-05 | ETF 折溢價：每日計算、列表、歷史走勢、無 NAV 者標示不適用 | P1 |
| FR-06 | 量化預測區間帶（5/20 日）＋AI 三情境呈現 | P1 |
| FR-07 | 模擬交易：虛擬帳戶、AI 自動下單、事件溯源交易日誌、權益曲線 | P1 |
| FR-08 | AI 用量儀表板：各模型當日已用/剩餘請求數 | P1 |
| FR-09 | 策略回測（規則策略，不進 AI） | P2 |
| FR-10 | 價格/折溢價警示 | P2 |

## 5. 非功能需求（NFR）

| 編號 | 需求 | 標準 |
|------|------|------|
| NFR-01 | 成本 | 營運成本 $0；AI 全走免費額度 |
| NFR-02 | AI 額度安全 | 任何情況不得超打免費額度：節流器＋當日快取；例行分析額度不足時明確失敗，不改用非結構化模型 |
| NFR-03 | 效能 | 頁面首次載入 < 3s；圖表資料 API < 500ms（讀本地快取，不現抓外部 API） |
| NFR-04 | 可靠性 | 外部資料源失敗自動重試 3 次＋換備援源；排程失敗記 log 且不影響既有資料 |
| NFR-05 | 資料正確性 | 外部資料入庫前經 schema 驗證；折溢價/報酬率計算有單元測試覆蓋 |
| NFR-06 | 可測試性 | 指標計算、撮合引擎、折溢價計算 80%+ 覆蓋率 |
| NFR-07 | 可攜性 | Docker Compose 一鍵啟動；本機與雲端部署同一套映像 |
| NFR-08 | 免責 | 所有 AI 建議與預測 UI 皆標示「僅供參考，非投資建議」 |
| NFR-09 | 背景工作可靠性 | 長任務寫入 DB queue，支援 lease/heartbeat、程序重啟恢復、冪等與人工重試 |
| NFR-10 | 可觀測性 | liveness/readiness 分離；回應包含 request ID；資料狀態分列 news/routine/trade 與最近成功工作 |
| NFR-11 | 安全與供應鏈 | production fail-closed、敏感資訊遮罩、安全標頭、鎖定依賴、依賴掃描與非 root container |

## 6. 外部介面與依賴

| 依賴 | 用途 | 免費限制 | 備援 |
|------|------|---------|------|
| FinMind API | 台股日線、法人、ETF 淨值 | 免費 token 每小時請求上限 | TWSE/TPEX OpenAPI |
| TWSE/TPEX OpenAPI | 台股官方資料、折溢價 | 無 key，禮貌性限流 | FinMind |
| yfinance | 美股日線、ETF NAV | 非官方，介面可能變動 | Stooq |
| Gemini API | AI 分析（3.1 Flash Lite 500 RPD／3.5 Flash 20 RPD） | 額度依帳號儀表板 | 例行分析只用 Flash Lite；重要分析可由 3.5 降級至 Flash Lite |
| Antigravity（Interactions API） | 新聞/事件研究 | 100 RPD | 跳過該日新聞面 |

## 7. 限制與假設

- 單人使用，無認證系統（部署於本機或私有網路；若上公網需加 Basic Auth）
- 日線級資料，非即時報價；模擬成交採隔日開盤價，不模擬盤中滑價
- 台股美股各自獨立記帳（TWD/USD），不做匯率換算
- AI 為 preview/免費服務，品質與可用性不保證 → 系統設計上模擬交易有硬風控，AI 掛掉只是「當日不動作」，不會壞帳
